Reach supports the following data types in index.rsh files. Use Type operators and functions to inspect the data types of values.

Address

An Address represents an account address. The Address format differs slightly depending on the consensus network. The following example reports the account address of a participant:

  1. // index.rsh
  2. const myInteract = {
  3.   reportAddress: Fun([Address], Null)
  4. };
  5. M.interact.reportAddress(this);

Here is one implementation of the corresponding frontend interact object:

  1. // index.mjs
  2. const myInteract = {
  3.   reportAddress: (v) => { console.log(v); }
  4. };

And, here is the output:

  1. # REACH_CONNECTOR_MODE=ALGO-devnet
  2. 0x5b26ec467ab3e54c793315b548923ec376624d3d29e70083bd2555ef24da8383
  3.   
  4. # REACH_CONNECTOR_MODE=CFX-devnet
  5. NET999:ACD8UE5KYBVG62MB9Z8R20CFBPHJ632ESJTEKUZ31V
  6.   
  7. # REACH_CONNECTOR_MODE=ETH-devnet
  8. 0xD2651496526c4E93fc01A87F99bD38092A244FB5

The addressEq function compares two addresses. The following example compares the addresses of a participant and the contract:

  1. // index.rsh
  2. const myInteract = {
  3.   reportBool: (v) => { console.log(v); }
  4. };
  5. M.interact.reportBool(addressEq(getAddress(), this));

The operators ==, ===, !=, !== are also valid:

  1. // index.rsh
  2. M.interact.reportBool(getAddress() == this);

API

An API expression defines an API in the contract:

  1. export const main = Reach.App(() => {
  2.   const P = Participant('Participant', {});
  3.   const V = View('Viewer', {});
  4.   const A = API('Writer', {
  5.     writeN: Fun([UInt], State),
  6.     writeT: Fun([UInt], State),
  7.     writeB: Fun([UInt], State),
  8.     writeX: Fun([UInt], State)
  9.   });
  10.   deploy();
  11.   //...
  12.   exit();
  13. });

API arguments include a name and an interface comprised of methods available in frontends via the ctc.apis object. The return value is an object whose fields are the interface methods. The object may be used in the .api component of fork and parallelReduce expressions. Each method must occur exactly once in the entire program. See this example.

Array

An Array is an immutable, ordered list of values. The size of an array is fixed, and the values must be of the same type. Values may be referenced using a zero-based index. The following creates an array of type Array(UInt, 3) from a tuple:

  1. const a = array(UInt, [4, 6, 8]);
  2. const v = a[0]; // 4

The following creates an array of type Array(Object({price: UInt, quantity: UInt}), 3) from a tuple:

  1. const a = array(
  2.   Object({price: UInt, quantity: UInt}),
  3.   [
  4.     {price:1000, quantity:10},
  5.     {price:2000, quantity:20},
  6.     {price:3000, quantity:30}
  7.   ]
  8. );
  9. const p = a[0].price; // 1000

Array.all

This method returns true if every element in the array satisfies the given Bool function. Otherwise, it returns false.

  1. // index.rsh
  2. const myInteract = { reportBool: Fun([Bool], Null) };
  3. const a = array(UInt, [4, 6, 8]);
  4. M.interact.reportBool(Array.all(a, e => e > 0)); // true
  5. // or
  6. M.interact.reportBool(a.all(e => e > 0)); // true

Array.and

This method returns true if all elements in a Bool array are true. Otherwise, it returns false.

  1. // index.rsh
  2. const myInteract = { reportBool: Fun([Bool], Null) };
  3. const a = array(Bool, [true, false, true]);
  4. M.interact.reportBool(Array.and(a)); // false

Array.any

This method returns true if at least one element in the array satisfies the given Bool function. Otherwise, it returns false.

  1. // index.rsh
  2. const myInteract = { reportBool: Fun([Bool], Null) };
  3. const a = array(UInt, [4, 6, 8]);
  4. M.interact.reportBool(Array.any(a, e => e > 7)); // true
  5. // or
  6. M.interact.reportBool(a.any(e => e > 7)); // true

Array.average

This method returns the mean of an array of UInts.

  1. // index.rsh
  2. const myInteract = { reportUInt: Fun([UInt], Null) };
  3. const a = array(UInt, [4, 6, 8]);
  4. M.interact.reportUInt(Array.average(a)); // 6
  5. // or
  6. M.interact.reportUInt(a.average()); // 6

Array.concat

This method returns a new array representing the concatenation of the two given arrays.

  1. // index.rsh
  2. const myInteract = { reportUIntArray: Fun([Array(UInt, 6)], Null) };
  3. const a = array(UInt, [4, 6, 8]);
  4. const b = array(UInt, [10, 12, 14]);
  5. M.interact.reportUIntArray(Array.concat(a, b)); // array(UInt, [4, 6, 8, 10, 12, 14])
  6. // or
  7. M.interact.reportUIntArray(a.concat(b)); // array(UInt, [4, 6, 8, 10, 12, 14])

Array.count

This method returns the number of elements in the array that satisfy the given Bool function.

  1. // index.rsh
  2. const myInteract = { reportUInt: Fun([UInt], Null) };
  3. const a = array(UInt, [4, 6, 8]);
  4. M.interact.reportUInt(Array.count(a, e => e > 5)); // 2
  5. // or
  6. M.interact.reportUInt(a.count(e => e > 5)); // 2

Array.elemType

This method returns the type of element that the array contains.

  1. // index.rsh
  2. const a = array(UInt, [4, 6, 8]);
  3. const t = Array.elemType(a);
  4. // or
  5. const t = a.elemType;

Array.empty

Array.empty is an array with no elements. It is the identity element of Array.concat. It may also be written Array_empty.

Array.find

This method returns the first element in the array that satisfies the given Bool function. The return value is of type Data.

  1. // index.rsh
  2. const myInteract = { reportData: Fun([Data({"None": Null, "Some": UInt})], Null) };
  3. const a = array(UInt, [4, 6, 8]);
  4. M.interact.reportData(Array.find(a, e => e > 3)); // Some,4
  5. // or
  6. M.interact.reportData(a.find(e => e > 3)); // Some,4

Array.findIndex

This method returns the index of the first element in the array that satisfies the given Bool function. The return value is of type Data.

  1. // index.rsh
  2. const myInteract = { reportData: Fun([Data({"None": Null, "Some": UInt})], Null) };
  3. const a = array(UInt, [4, 6, 8]);
  4. M.interact.reportData(Array.findIndex(a, e => e > 5)); // Some,1
  5. // or
  6. M.interact.reportData(a.findIndex(e => e > 5)); // Some,1

Array.forEach

This method iterates the specified function over the elements of the array and returns Null. It is often used to transfer funds to accounts:

  1. // index.rsh
  2. // Assume that sale.arbitrator is Array(Address, 3)
  3. const arbFee = (buyerPmt * (2 / 100)) / Array.length(sale.arbitrator);
  4. Array.forEach(sale.arbitrator, (a) => {
  5.   transfer(arbFee).to(a);
  6. });
  7. transfer(balance()).to(Owner);

Array.includes

This method returns true if the array contains the specified element. Otherwise, it returns false.

  1. // index.rsh
  2. const myInteract = { reportBool: Fun([Bool], Null) };
  3. const a = array(UInt, [4, 6, 8]);
  4. M.interact.reportBool(Array.includes(a, 4)); // true
  5. // or
  6. M.interact.reportBool(a.includes(4)); // true

Array.indexOf

This method returns the index of the first element in the array that equals the given value. The return value is of type Data.

  1. // index.rsh
  2. const myInteract = { reportData: Fun([Data({"None": Null, "Some": UInt})], Null) };
  3. const a = array(UInt, [4, 6, 8]);
  4. M.interact.reportData(Array.indexOf(a, 6)); // Some,1
  5. // or
  6. M.interact.reportData(a.indexOf(6)); // Some,1

Array.iota

This method returns an array of the specified length where each element is the same as its index.

  1. // index.rsh
  2. const myInteract = { reportUIntArray: Fun([Array(UInt, 3)], Null), };
  3. M.interact.reportUIntArray(Array.iota(3)); // array(UInt, [0, 1, 2])

Array.length

This method returns the length of the array.

  1. // index.rsh
  2. const myInteract = { reportUInt: Fun([UInt], Null) };
  3. const a = array(UInt, [4, 6, 8]);
  4. M.interact.reportUInt(Array.length(a)); // 3
  5. // or
  6. M.interact.reportUInt(a.length); // 3

Array.map

This method applies the given function to each element of the given array, and returns a new same-length array containing the transformed values.

  1. // index.rsh
  2. const myInteract = { reportUIntArray: Fun([Array(UInt, 3)], Null) };
  3. const a = array(UInt, [4, 6, 8]);
  4. M.interact.reportUIntArray(Array.map(a, e => e + 1)); // array(UInt, [5, 7, 9])
  5. // or
  6. M.interact.reportUIntArray(a.map(e => e + 1)); // array(UInt, [5, 7, 9])
  7. // also
  8. M.interact.reportUIntArray(Array.iota(3).map(a, add)); // array(UInt, [4, 7, 10])

Array.mapWithIndex

This method is similar to Array.map except that it provides the function with the element's index, too.

  1. // index.rsh
  2. const myInteract = { reportObjectArray: Fun([Array(Object({value: UInt, index: UInt}), 3)], Null) };
  3. const a = array(UInt, [4, 6, 8]);
  4. M.interact.reportObjectArray(Array.mapWithIndex(a, (e, i)  => {return {"value": e + 1, "index": i}}));
  5. // or
  6. M.interact.reportObjectArray(a.mapWithIndex((e, i)  => {return {"value": e + 1, "index": i}}));

Here is one implementation of the corresponding frontend interact object:

  1. // index.mjs
  2. reportObjectArray: (v) => { console.log(v); }

Here is the output:

  4.     value: BigNumber { _hex: '0x05', _isBigNumber: true }
  5.   },
  6.   {
  7.     index: BigNumber { _hex: '0x01', _isBigNumber: true },
  8.     value: BigNumber { _hex: '0x07', _isBigNumber: true }
  9.   },
  10.   {
  11.     index: BigNumber { _hex: '0x02', _isBigNumber: true },
  12.     value: BigNumber { _hex: '0x09', _isBigNumber: true }
  13.   }
  14. ]

Array.product

This method returns the product of an array of unsigned integers:

  1. // index.rsh
  2. const myInteract = { reportUInt: Fun([UInt], Null) };
  3. const a = array(UInt, [4, 6, 8]);
  4. M.interact.reportUInt(Array.product(a)); // 192
  5. // or

Array.reduce

Starting with the specified value, this method returns the left fold of the given function over the array:

  1. // index.rsh
  2. const myInteract = { reportUInt: Fun([UInt], Null) };
  3. const a = array(UInt, [4, 6, 8]);
  4. M.interact.reportUInt(Array.reduce(a, 0, (x, total) => (total + x))); // 18
  5. // or
  6. M.interact.reportUInt(Array.reduce(a, 0, add)); // 18
  7. // or
  8. M.interact.reportUInt(a.reduce(0, (x, total) => (total + x))); // 18
  9. // or
  10. M.interact.reportUInt(a.reduce(0, add)); // 18
  11. // also
  12. M.interact.reportUInt(Array.iota(3).reduce(a, 0, (x, y, total) => (total + x + y))); // 21

Array.reduceWithIndex

This method is similar to Array.reduce except that it provides the function with the element's index, too.

Array.replicate

This method creates an array of the specified length where each element is the specified value:

  2. const myInteract = { reportUIntArray: Fun([Array(UInt, 3)], Null) };

Array.withIndex

This method creates a new same-length array where each element pairs the corresponding original element with its index.

  1. // index.rsh
  2. const myInteract = { reportTupleArray: Fun([Array(Tuple(UInt, UInt), 3)], Null) };
  3. const a = array(UInt, [4, 6, 8]);

Array.zip

Given two same-length arrays, this method creates a new same-length array where each element pairs the elements of the original arrays.

  1. // index.rsh
  2. const myInteract = { reportTupleArray: Fun([Array(Tuple(UInt, UInt), 3)], Null) };
  3. const a = array(UInt, [4, 6, 8]);
  4. const b = array(UInt, [10, 12, 14]);
  5. M.interact.reportTupleArray(Array.zip(a, b)); // array(Tuple(UInt, UInt), [[4, 10], [6, 12], [8, 14]])
  6. // or
  7. M.interact.reportTupleArray(a.zip(b)); // array(Tuple(UInt, UInt), [[4, 10], [6, 12], [8, 14]])

Bool

A Bool is a boolean which may be true or false.

Bytes

Bytes is a specialized (immutable, fixed-length) array of characters:

  1. // index.rsh
  2. const ar = 'فراشة';          // Bytes(10)
  3. const de = 'Schmetterling';  // Bytes(13)
  4. const en = 'butterfly';      // Bytes(9)
  5. const fr = 'papillon';       // Bytes(8)
  6. const ga = 'féileacán';      // Bytes(11)
  7. const he = 'פַּרְפַּר';           // Bytes(18)
  8. const vi = 'Con bướm';       // Bytes(11)
  9. const zh = '蝴蝶';            // Bytes(6)

Two Bytes of different lengths are not interchangeable. Consider the following:

  1. // index.rsh
  2. const myInteract = { reportBytes: Fun([Bytes(24)], Null) };
  3. M.interact.reportBytes('butterfly'); // Bytes(9)

Because 'butterfly' is of type Bytes(9) and reportBytes requires an argument of type Bytes(24), the compiler, encountering this code, will generate an error similar to the following:

  1. reachc: error[RE0088]: These types are mismatched: Bytes(9) vs Bytes(24)

To match the two types, use the pad function:

  1. // index.rsh
  2. const myInteract = { reportBytes: Fun([Bytes(24)], Null) };
  3. M.interact.reportBytes(Bytes(24).pad('butterfly'));

Here is one implementation of the corresponding frontend interact object:

  1. // index.mjs
  2. const myInteract = { reportBytes: (v) => { console.log(`Length is ${v.length}. Value is ${v}.`); }};

Note that the lengths vary:

  1. Length is 19. Value is فراشة.
  2. Length is 24. Value is Schmetterling.
  3. Length is 24. Value is butterfly.
  4. Length is 24. Value is papillon.
  5. Length is 22. Value is féileacán.
  6. Length is 15. Value is פַּרְפַּר.
  7. Length is 21. Value is Con bướm.
  8. Length is 20. Value is 蝴蝶.

transfer

A transfer expression transfers the specified number of tokens from the contract account to the specified participant account. Here is the declaration:

  1. transfer(AMOUNT_EXPR).to(ADDR_EXPR)
  2. // or
  3. transfer(AMOUNT_EXPR, TOKEN_EXPR).to(ADDR_EXPR)
  • AMOUNT_EXPR is an unsigned integer specifying the number of tokens to transfer from the contract. This amount must evaluate to less than or equal to the balance of network tokens in the contract account.
  • ADDR_EXPR is the target address for the transfer.
  • TOKEN_EXPR is a Token type. This argument is optional.

Here is an example:

  1. transfer(10).to(Alice);

Data

Use Data to define a data type (e.g. Shape):

  1. const Shape = Data({
  2.   Circle: Object({ radius: UInt }),
  3.   Square: Object({ side: UInt }),
  4.   Rect: Object({ width: UInt, height: UInt })
  5. });

In the example, the Shape data type has three variants: Circle, Square, and Rect. Create data instances like this:

  1. const circle = Shape.Circle({ radius: 7 });
  2. const square = Shape.Square({ side: 5 });
  3. const rect = Shape.Rect({ width: 3, height: 6 });

Data instances (e.g. circle, square, rect) are consumed by switch statements and match expressions.

Digest

A Digest is a cryptographic hash of a value:

  1. // index.rsh
  2. const sellerInteract = {
  3.   password: Bytes(24), // "sellerPwd123$"
  4.   reportDigest: Fun([Digest], Null)
  5. };
  7. export const main = Reach.App(() => {
  8.   const S = Participant('Seller', sellerInteract);
  9.   const B = Participant('Buyer', {});
  10.   deploy();
  11.   S.publish();
  12.   commit();
  13.   B.publish();
  14.   commit();
  16.   S.only(() => {
  17.     const _pwd = digest(interact.password);
  18.     interact.reportDigest(_pwd);
  19.     const pwdDigest = declassify(_pwd);
  20.   });
  21.   S.publish(pwdDigest);
  22.   commit();
  24.   exit();
  25. });
  • Line 3: Assume that the frontend furnishes "sellerPwd123$" as the clear-text password.
  • Line 17: digest converts the password to a hash (0x51db28...).
  • Line 18: reportDigest passes the hash value to the frontend to demonstrate the use of the Digest type and to show you one way to inspect the hash value. Normally, you would not pass a hash value to a frontend.

The hash value for "sellerPwd123$" is the following:

  1. 0x51db281cfa433bd72491c79c46d0a2d1da211fda094e5904f2d3d77bbd51a2d5

You cannot un-digest a hash value to reveal the original string, but, you can compare hashed values:

  1. require( pwdDigest == digest(buyerPassword) );

Either

Either is a built-in Data type with two variants (i.e. Left and Right). It is implemented as a function that assigns type A to Left and type B to Right:

  1. export const Either = (A, B) => Data({Left: A, Right: B});

Either is useful for representing either the successful (i.e. Right) or the unsuccessful (i.e. Left) state of a return value. Consider, for example, a function called addEvenNumbers which only adds even numbers, and returns a value of type Either containing a sum in the Right variant if both arguments are even, and an object (containing the first offending argument and a message) in the Left variant if either argument is odd:

  1. // index.rsh
  2. function addEvenNumbers(a, b) {
  3.   const answer = Either(Object({ arg: UInt, msg: Bytes(40) }), UInt);
  4.   if(a % 2 != 0) {
  5.     return answer.Left({arg: a, msg: Bytes(40).pad('First arg must be an even number.')});
  6.   } else if (b % 2 != 0) {
  7.     return answer.Left({arg: b, msg: Bytes(40).pad('Second arg must be an even number.')});
  8.   }
  9.   return answer.Right(a + b);
  10. }

One way to invoke addEvenNumbers and deal with the return value is like this:

  1. // index.rsh
  2. const myInteract = {
  3.   reportLeft: Fun([Object({"arg": UInt, "msg": Bytes(40)})], Null),
  4.   reportRight: Fun([UInt], Null)
  5. };
  7. const myEither = addEvenNumbers(6, 4);
  8. M.only(() => { 
  9.   either(myEither, interact.reportLeft, interact.reportRight);
  10. }

Reach provides the following methods for processing Either types:

either

This method determines whether an argument of type Either is a Left or Right variant, and then invokes the relevant function passing the relevant variant as an argument:

  1. // index.rsh
  2. const myInteract = {
  3.   reportLeft: Fun([Object({"arg": UInt, "msg": Bytes(40)})], Null),
  4.   reportRight: Fun([UInt], Null)
  5. };
  7. const myEither = addEvenNumbers(6, 4);
  8. M.only(() => { 
  9.   either(myEither, interact.reportLeft, interact.reportRight);
  10. }

fromLeft

This method returns the Left variant or, if the Left variant does not exist, a default value:

  1. // index.rsh
  2. const myInteract = {
  3.   reportLeft: Fun([Object({"arg": UInt, "msg": Bytes(40)})], Null)
  4. };
  6. const myEither = addEvenNumbers(5, 4);
  7. S.only(() => { 
  8.   interact.reportLeft(fromLeft(myEither, { arg: 0, msg: Bytes(40).pad('No error.')}));
  9. });

fromRight

This method returns the Right variant or, if the Right variant does not exist, a default value:

  1. // index.rsh
  2. const myInteract = {
  3.   reportRight: Fun([UInt], Null)
  4. };
  6. const myEither = addEvenNumbers(5, 4);
  7. S.only(() => { 
  8.   interact.reportRight(fromRight(myEither, 0));
  9. });

isLeft

This method returns true if the Left variant exists:

  1. // index.rsh
  2. const myInteract = {
  3.   reportBool: Fun([Bool], Null)
  4. };
  6. const myEither = addEvenNumbers(5, 4);
  7. S.only(() => { 
  8.   interact.reportBool(isLeft(myEither));
  9. });

isRight

This method returns true if the Right variant exists:

  1. // index.rsh
  2. const myInteract = {
  3.   reportBool: Fun([Bool], Null)
  4. };
  6. const myEither = addEvenNumbers(5, 4);
  7. S.only(() => { 
  8.   interact.reportBool(isRight(myEither));
  9. });

left

This method assigns a value to the Left variant:

  1. // index.rsh
  2. const answer = Either(Object({ arg: UInt, msg: Bytes(40) }), UInt);
  3. answer.Left({arg: a, msg: Bytes(40).pad('First arg must be an even number.')});

right

This method assigns a value to the Right variant:

  1. // index.rsh
  2. const answer = Either(Object({ arg: UInt, msg: Bytes(40) }), UInt);
  3. answer.Right(a + b);

FixedPoint

A FixedPoint is an object { sign: Bool, i: { scale: UInt, i: UInt} } where scale is 1, 10, 100, 1000, etc., and i is the underlying unsigned integer. The value of a fixed-point number is (i / scale) * sign. It is used to represent numbers that have a fixed number of digits after the decimal.

Map

A Map is a means of associating participants with values of a given type:

Participant Value
Participant 1 200
Participant 2 400
Participant 3 100
Participant 4 125

A Map represents participants by account address:

Participant Value
0x1234abcd0 200
0x1234abcd1 400
0x1234abcd2 100
0x1234abcd3 125

The Map constructor requires a type:

  1. const m = new Map(UInt);

Use [] to reference (Line 10) and dereference (Line 11):

  1. // index.rsh
  2. const sellerInteract = {
  3.   reportData: Fun([Data({"None": Null, "Some": UInt})], Null),
  4. };
  5. export const main = Reach.App(() => {
  6.   const S = Participant('Seller', sellerInteract);
  7.   deploy();
  8.   // ...
  9.   const m = new Map(UInt); // Address <> UInt
  10.   m[S] = 256;              // reference
  11.   const v = m[S];          // dereference
  12.   S.interact.reportData(v);
  13.   // ...
  14. });

In the example above, const v is not a UInt. It is a Data({"None": Null, "Some": UInt}) because m[S] allows for the possibility that the specified participant address is not in the Map. See Maybe.

Map is a Foldable container, so it includes all the Foldable methods, but these methods are only valid within the invariant of a while loop. Consider the following:

  1. const ctMap = new Map(UInt);
  2. // ...
  3. .invariant( balance() == sum && balance() == ctMap.sum() )
  4. .while( !stop && balance() < p.goal )
  5. // ...

Line 3 shows Map.sum within the loop invariant. See Loop Invariant for a more detailed example.

Maybe

Maybe is a built-in Data type with two variants (i.e. None and Some). It is implemented as a function that assigns a particular type A to the Some variant:

  1. export const Maybe = (A) => Data({ None: Null, Some: A });

Maybe is useful when a particular action may or may not return a valid value. Dereferencing a Map is one example:

  1. const myMap = new Map(UInt);
  2. // ...
  3. const myMaybe = myMap[Seller]; // Maybe

In the example above, Seller may or may not be a valid address in myMap, so myMap[Seller] returns a Maybe to cover both possibilities. Programs use the functions fromSome, isNone, and isSome to evaluate Maybe values as described below.

Programs create Maybe values like this:

  1. const mUInt = Maybe(UInt);
  2. const success = mUInt.Some(256);
  3. const failure = mUInt.None();

fromSome

This method takes a Maybe value and a default value as arguments. It returns the Some variant is it exists. Otherwise, it returns the default value:

  1. const v = fromSome(myMaybe, 0);
  2. if(v) {
  3.   // ...
  4. } else {
  5.   // ...
  6. }

isNone

This method returns true if the specified Maybe argument is None:

  1. if( isNone( myMaybe ) ) {
  2.   // ...
  3. }

isSome

This method returns true if the specified Maybe argument is Some:

  1. if( isSome( myMaybe ) ) {
  2.   // ...
  3. }

Object

An Object is a container of key-value pairs where keys are identifiers (e.g. name) or strings (e.g. 'name'), and values are expressions:

  1. // index.rsh
  2. const product = {
  3.   name: Bytes(24).pad('Carrots'),
  4.   unit: Bytes(10).pad('Bunch'),
  5.   units: Bytes(10).pad('Bunches'),
  6.   price: 100
  7. };

A Reach backend might pass product to a frontend like this:

  1. const myInteract = {
  2.   reportObject: Fun([Object({name: Bytes(24), price: UInt, unit: Bytes(10), units: Bytes(10)})], Null)
  3. };
  4. M.interact.reportObject(product);

Objects have the following methods:

Object.has

This method returns true if the specified object has the specified field. Otherwise, it returns false:

  1. const myInteract = { reportBool: Fun([Bool], Null) };
  2. const product = {
  3.   name: Bytes(24).pad('Carrots'),
  4.   unit: Bytes(10).pad('Bunch'),
  5.   units: Bytes(10).pad('Bunches'),
  6.   price: 100
  7. };
  8. M.interact.reportBool(Object.has(product, 'name')); // true

Object.set

This method returns a new object identical to the specified one except that the specified field (which may or may not exist in the original object) is set to the specified value. The following example resets an existing field:

  1. const myInteract = {
  2.   reportObject: Fun([Object({name: Bytes(24), price: UInt, unit: Bytes(10), units: Bytes(10)})], Null)
  3. };
  4. const product = {
  5.   name: Bytes(24).pad('Carrots'),
  6.   unit: Bytes(10).pad('Bunch'),
  7.   units: Bytes(10).pad('Bunches'),
  8.   price: 100
  9. };
  10. const product2 = Object.set(product, "price", 200);
  11. M.interact.reportObject(product2);

The following example adds a new field:

  1. const sellerInteract = {
  2.   reportObject: Fun([Object({name: Bytes(24), price: UInt, unit: Bytes(10), units: Bytes(10), color: Bytes(12)})], Null)
  3. };
  4. const product = {
  5.   name: Bytes(24).pad('Carrots'),
  6.   unit: Bytes(10).pad('Bunch'),
  7.   units: Bytes(10).pad('Bunches'),
  8.   price: 100
  9. };
  10. const product2 = Object.set(product, "color", Bytes(12).pad("orange"));
  11. S.interact.reportObject(product2);

Object.setIfUnset

This method is similar to Object.set. However, Object.setIfUnset will not overwrite an existing property value. It will only add a new property and value to the specified object.

Token

The Token expression mints a new non-network token. Here is the declaration:

  1. new Token(PARAMS)

  • PARAMS is an object with the keys in the table below.

    Key Type Default
    name Bytes(32) empty
    symbol Bytes(8) empty
    url Bytes(96) empty
    metadata Bytes(32) empty
    supply UInt UInt.max
    decimals UInt ALGO = 6; CFX and ETH = 18

This returns a Token value and deposits a supply amount of the new non-network tokens into the contract account associated with the DApp. These tokens must be destroyed by the end of the DApp. A token has the following methods:

  • Token.burn(tok, amt) or tok.burn(amt), where tok is a Token value and amt is a UInt value, may be used to burn tokens in the contract account, meaning that they are utterly destroyed and can never be recovered.

  • Token.destroy(tok) or tok.destroy(), where tok is a Token value, may be used to destroy the token so that it may never be used again by any users on the consensus network. This must be called before the application exits.

  • Token.destroyed(tok) or tok.destroyed(), where tok is a Token value, returns whether destroy has been called on tok yet.

  • Token.supply(tok) or tok.supply(), where tok is a Token value, may be used to query the current supply of tokens, i.e. the number of tokens which have not been burnt.

Here is an example:

  1. require(supply >= 2 * amt);
  2. const tok = new Token({ name, symbol, url, metadata, supply, decimals });
  3. transfer(amt, tok).to(who);
  4. tok.burn(amt);
  5. assert(tok.supply() == supply - amt);
  6. tok.burn();
  7. assert(tok.destroyed() == false);
  8. tok.destroy();

Tuple

A Tuple is an immutable, ordered list of values. The size of a tuple is fixed. The values may be different types. Values may be referenced using a zero-based index. The following creates a tuple of type Tuple(UInt, UInt, UInt):

  1. const t = [5, 10, 15];
  2. const v = t[2]; // 15

The following creates a tuple of type Tuple(UInt, Object({price: UInt, quantity: UInt}), UInt):

  1. const t = [5, { price: 1000, quantity: 40 }, 15];
  2. const p = t[1].price; // 1000
  3. const v = t[2];       // 15

The following creates a tuple of type Tuple(UInt, UInt, UInt, UInt, Object({price: UInt, quantity: UInt}), UInt):

  1. const t_temp = [5, { price: 1000, quantity: 40 }, 15];
  2. const t = [5, 10, 15, ...t_temp];
  3. const p = t[4].price; // 1000
  4. const v = t[5];       // 15

Tuple.length

Tuple.length returns the length of the tuple:

  1. const lenA = Tuple.length([5, 10, 15]); // 3
  2. // or
  3. const lenB = [5, 10, 15].length; // 3

Tuple.set

Tuple.set returns a new tuple identical to the original except that the value at the specified index is the new value:

  1. const tA = Tuple.set([5, 10, 15], 1, 44);
  2. const vA = tA[1]; // 44
  3. // or
  4. const tB = [5, 10, 15].set(1, 44);
  5. const vB = tB[1]; // 44

sqrt

The sqrt function finds the square root of value, iterating iterations number of times.

  1. sqrt(value, iterations) // sqrt(UInt, UInt)

This function utilizes the Babylonian Method for computing the square root. iterations must be known at compile time. When performing 5 iterations, the algorithm can reliably calculate the square root up to 32 squared, or 1,024. When performing 10 iterations, the algorithm can reliably calculate the square root up to 580 squared, or 336,400.

pow

The pow function calculates the approximate value of raising base to power, iterating iterations number of times.:

  1. pow(base, power, iterations) // pow(UInt, UInt, UInt)

iterations must be known at compile time. Six iterations provides enough accuracy to calculate up to 263.
